# Матрица Sourcing Strategy (Продуктовый подход)

В результате аналитики ИТ-ландшафта VitaStore схожие по функционалу решения были объединены в **целевые продукты**. Это позволяет сократить число разрозненных ИТ-систем и сформировать понятную стратегию Buy vs Build.

| Целевой продукт / Решение                                                   | Охваченные возможности (L2)                                                                                                                             | Рекомендуемая модель (SaaS / PaaS / In-House / On-Premise) | Обоснование выбора и состав решения                                                                                                                                                                                                                                                                                                                                                 |
| :----------------------------------------------------------------------------| :--------------------------------------------------------------------------------------------------------------------------------------------------------| :-----------------------------------------------------------| :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Cloud POS с интегрированным эквайрингом (Облачная касса и прием оплаты)** | Кассовое обслуживание (POS), прием оплаты (эквайринг), поддержка платежных терминалов (драйверы), локальные расчеты, печать ценников, автообновление ПО | **SaaS (Buy)**                                             | **Единое облачное решение**. Тонкий клиент на планшетах со встроенным платежным шлюзом, поддерживающим любые типы банковских терминалов (включая ARCUS-2). Полностью ликвидирует локальные серверы в подсобках магазинов, убирает затраты на покупку локальных лицензий Windows/SQL (TCO снижается на 40%) и позволяет запустить кассу за 5 дней без написания кастомных драйверов. |
| **Promo & Loyalty Platform (Платформа лояльности и промо)**                 | Управление промо-акциями, лояльность на кассах, применение скидок                                                                                       | **In-House / PaaS (Build)**                                | **Собственная разработка**. Объединяет центральный Promo Engine и новую оцифрованную систему лояльности (замена тетрадкам). Кассовое ПО должно кэшировать правила скидок локально, чтобы лояльность и промо-акции бесперебойно работали при обрыве связи с HQ.                                                                                                                      |
| **Smart Fulfillment System (Платформа логистики)**                          | Умное распределение заказов (Click & Collect)                                                                                                           | **In-House / PaaS (Build)**                                | **Собственная разработка**. Работает в тесной синергии с физической сетью из 500+ магазинов, обеспечивая выдачу товаров со склада максимально близко к покупателю. Является главным конкурентным отличием VitaStore.                                                                                                                                                                |
| **M&A SKU Matching Service (Сервис маппинга товаров)**                      | Сопоставление баз товаров, подготовка позиций при слияниях (M&A)                                                                                        | **PaaS (Build / Hybrid)**                                  | **Собственная/гибридная микросервисная разработка**. Облачный сервис автоматического нечеткого сопоставления номенклатурных позиций (fuzzy matching). Позволяет товароведу в HQ через веб-интерфейс подтверждать только спорные позиции (Human-in-the-Loop). Сокращает ручной маппинг с 2 недель до нескольких часов.                                                               |
| **Core ERP (Центральная ERP)**                                              | Бухучет и отчетность, ценообразование, кадровый учет (HR), найм и увольнение                                                                            | **On-Premise (Keep As-Is)**                                | **Существующая SAP ERP**. Учетное ядро полностью выполняет свои задачи. Изменения и миграция нецелесообразны из-за рисков и стоимости. Интегрируется с облачными продуктами через шлюз API.                                                                                                                                                                                         |
| **Cloud Managed Network & DB (Облачная база и связь)**                      | Управление остатками (Cloud DB), серверное и сетевое обслуживание                                                                                       | **PaaS (Migrate to Cloud)**                                | **Управляемая инфраструктура**. Локальные базы остатков переносятся в управляемую облачную СУБД (синхронизация по API в реальном времени взамен ночного FTP). Внедряется централизованная сеть для удаленного контроля без выезда инженеров.                                                                                                                                        |
